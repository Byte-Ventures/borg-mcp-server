Client Facade: borg server

Client-owned copy only. Server lifecycle output is passed through unchanged.

Help

Usage: borg server <command> [arguments]

Commands:
  setup    Prepare local server identity and data; does not start the server.
  start    Start the verified server in the foreground.
  status   Report verified runtime evidence.
  update   Verify and activate a local server artifact.
  invite   Create a single-use invitation in an interactive terminal.

Run borg server <command> --help for server command options.

Unsupported Command

Unknown server command: <command>.
Available commands: setup, start, status, update, invite.
Next: run borg server --help.

Missing Executable

Local server command is unavailable: borg-mcp-server was not found.
Next: install a verified borgmcp-server release, then rerun
borg server <command>.
No checkout fallback is attempted.

Server Command Startup Failure

Local server command could not be started.
Next: check local permissions and system resources, then rerun
borg server <command>.
No server command was started.

Use this only when process creation fails for a reason other than ENOENT. It exits 1. ENOENT alone uses Missing Executable and exits 127. Render command tokens as inert text, capped at 80 Unicode code points total; replace each Unicode control code point with ? and reserve the final three code points for ... when truncated.

Invite

borg server invite forwards to the server. Interactive server output is passed through unchanged:

Invitation (single-use; shown once): <invitation>
Share it only with the intended recipient.

Non-TTY: Invitation creation requires an interactive terminal.

Non-TTY

Use the same bounded plain text. Do not emit ANSI, JSON, a stack trace, a searched path, raw spawn errors, or a checkout hint for facade errors.
